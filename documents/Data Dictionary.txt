## SALES_FACT Table

**Purpose:** Core transaction table containing individual sales line items

**Record Count:** 395,000 transactions (after cleaning)

**Time Period:** December 1, 2010 - December 9, 2011

**Primary Key:** InvoiceNo + StockCode (composite)

### Column Definitions

| Column Name | Data Type | Description | Example | Notes |
|-------------|-----------|-------------|---------|-------|
| **InvoiceNo** | VARCHAR(10) | Unique transaction identifier | B01234 | Primary Key, identifies each invoice |
| **StockCode** | VARCHAR(20) | Product identifier | 10001 | Foreign Key to dim_products.StockCode |
| **Description** | VARCHAR(255) | Product name/description | Paper Craft Little | Descriptive product name |
| **Quantity** | INTEGER | Units sold in transaction | 2, 81, 54 | Positive numbers only (returns removed) |
| **InvoiceDate** | DATETIME | Date and time of transaction | 2010-12-01 08:26:00 | Converted to TransactionDate (date only) |
| **UnitPrice** | DECIMAL(10,2) | Price per unit in currency | 10.50, 12.99, 15.50 | Greater than 0 (promotions removed) |
| **TotalRevenue** | DECIMAL(12,2) | Calculated: Quantity × UnitPrice | 21.00, 1,051.00 | Automatically calculated |
| **CustomerID** | INTEGER | Unique customer identifier | 12345, 67890 | Foreign Key to dim_customers.CustomerID |
| **Country** | VARCHAR(50) | Customer's country | United Kingdom, France | 38 unique countries |
| **TransactionDate** | DATE | Date only (no time) | 2010-12-01 | Derived from InvoiceDate, Foreign Key to dim_calendar.Date |

### Data Characteristics

```
Revenue Range:
├─ Minimum: $0.01
├─ Maximum: $38,970.00
└─ Average: $385.50

Quantity Range:
├─ Minimum: 1 unit
├─ Maximum: 80,995 units
└─ Average: ~12 units per line

Temporal Coverage:
├─ Start: December 1, 2010
├─ End: December 9, 2011
└─ Total Days: 374 days
```

### Quality Notes

- ✅ No null values in key fields
- ✅ All quantities positive (returns filtered)
- ✅ All prices positive (promotions filtered)
- ✅ All customers have CustomerID
- ✅ Date range validated
- ❌ Removed records: 146,909 (26.7%)

## Relationships

### Foreign Key Relationships

**SALES_FACT → DIM_PRODUCTS**
```
Left (Many):     sales_fact.StockCode
Right (One):     dim_products.StockCode
Cardinality:     ∞:1 (many sales → one product)
Type:            Inner Join
Purpose:         Attach product details to sales
```

**SALES_FACT → DIM_CUSTOMERS**
```
Left (Many):     sales_fact.CustomerID
Right (One):     dim_customers.CustomerID
Cardinality:     ∞:1 (many sales → one customer)
Type:            Inner Join
Purpose:         Attach customer details to sales
```

**SALES_FACT → DIM_CALENDAR**
```
Left (Many):     sales_fact.TransactionDate
Right (One):     dim_calendar.Date
Cardinality:     ∞:1 (many sales → one date)
Type:            Inner Join
Purpose:         Attach date dimensions to sales
```

### Relationship Usage in Power BI

```
When filtering by dim_calendar.Quarter:
├─ Sales_fact is filtered by TransactionDate
├─ Dim_products filters by available StockCode
├─ Dim_customers filters by active CustomerID
└─ Result: All visuals show selected quarter only

When filtering by dim_customers.Country:
├─ Sales_fact is filtered by CustomerID
├─ Dim_calendar filters by dates with sales
├─ Dim_products filters by sold products
└─ Result: All visuals show selected country only
```

## Usage Examples

### Finding Product Performance

```sql
SELECT TOP 10
    p.ProductName,
    SUM(s.Quantity) as TotalUnits,
    SUM(s.TotalRevenue) as TotalRevenue
FROM sales_fact s
JOIN dim_products p ON s.StockCode = p.StockCode
GROUP BY p.ProductName
ORDER BY TotalRevenue DESC
```

### Finding Quarterly Trends

```sql
SELECT
    c.Quarter,
    c.Year,
    SUM(s.TotalRevenue) as QuarterlyRevenue,
    COUNT(DISTINCT s.InvoiceNo) as Orders
FROM sales_fact s
JOIN dim_calendar c ON s.TransactionDate = c.Date
GROUP BY c.Year, c.Quarter
ORDER BY c.Year, c.Quarter
```

### Finding Top Customers by Country

```sql
SELECT TOP 20
    cu.CustomerID,
    cu.Country,
    cu.PurchaseCount,
    cu.LifetimeValue
FROM dim_customers cu
WHERE cu.Country = 'United Kingdom'
ORDER BY cu.LifetimeValue DESC
```
