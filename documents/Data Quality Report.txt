## Data Issues Identified

### Issue #1: Negative Quantities (Returns/Cancellations)

**Problem Description:**
```
Records with negative Quantity values
Indicates returns, cancellations, or adjustments
Not part of main sales analysis
```

**Scale of Issue:**

```
Negative Quantity Records: 10,624 (2.0% of total)
├─ Quantity range: -1 to -80,995 units
├─ Revenue impact: ~$500K+ in returns/credits
├─ Distributed across: ~2,000 unique products
└─ Most common: -1 to -10 unit returns

Examples:
├─ InvoiceNo: A12345, Quantity: -2, Product: Paper Craft
├─ InvoiceNo: B67890, Quantity: -50, Product: Ceramic
└─ InvoiceNo: C11111, Quantity: -1000, Product: Bulk Items
```

**Why Removed:**
- Returns/cancellations distort sales analysis
- Negative revenue should not be combined with positive
- Separate analysis needed for return patterns
- Final "sales" dataset should show completed purchases

**Impact:**
- Removing these records eliminates $500K in net-offs
- Final revenue of $8.91M is NET revenue (after returns)
- This is CORRECT for business analysis
- More accurate than gross revenue

**Decision: ✅ REMOVED** (Correct approach)

### Issue #2: Zero Prices (Promotional Items)

**Problem Description:**
```
Records with UnitPrice = 0.00
Indicates free items or promotional giveaways
Inflate unit counts but add no revenue
```

**Scale of Issue:**

```
Zero Price Records: 2,515 (0.5% of total)
├─ Quantity: 10 to 5,000 units per record
├─ Total units affected: ~50,000 items
├─ Business impact: Free samples/promotions
└─ Most common products: Packing materials, samples

Examples:
├─ InvoiceNo: P11111, Quantity: 100, UnitPrice: $0.00
├─ InvoiceNo: P22222, Quantity: 50, UnitPrice: $0.00
└─ InvoiceNo: P33333, Quantity: 200, UnitPrice: $0.00
```

**Why Removed:**
- Zero-price items distort revenue analysis
- Giveaways should be tracked separately
- Revenue analysis is about actual sales, not free items
- Include would lower average order value artificially

**Impact:**
- Removes ~$0 in revenue (correct)
- Removes ~50K units from count
- Final unit count more accurate for "paid" sales
- This is CORRECT business approach

**Decision: ✅ REMOVED** (Correct approach)

### Step 6: Create Dimension Tables

**DIM_PRODUCTS:**
```sql
CREATE TABLE dim_products AS
SELECT DISTINCT
    StockCode,
    Description as ProductName,
    -- Add category assignment logic
    CASE WHEN Description LIKE '%craft%' THEN 'Crafts'
         WHEN Description LIKE '%ceramic%' THEN 'Ceramics'
         WHEN Description LIKE '%bag%' THEN 'Storage'
         -- ... more categories
         ELSE 'Other' END as Category
FROM sales_raw
```

**Result:** 4,223 unique products ✅

**DIM_CUSTOMERS:**
```sql
CREATE TABLE dim_customers AS
SELECT
    CustomerID,
    Country,
    COUNT(DISTINCT InvoiceNo) as PurchaseCount,
    SUM(TotalRevenue) as LifetimeValue,
    AVG(TotalRevenue) as AvgOrderValue
FROM sales_raw
GROUP BY CustomerID, Country
```

**Result:** ~4,372 unique customers ✅

**DIM_CALENDAR:**
```sql
CREATE TABLE dim_calendar AS
SELECT DISTINCT
    TransactionDate as Date,
    YEAR(TransactionDate) as Year,
    MONTH(TransactionDate) as Month,
    QUARTER(TransactionDate) as Quarter,
    DATENAME(WEEKDAY, TransactionDate) as DayName
FROM sales_raw
ORDER BY Date
```

**Result:** 365 unique dates ✅

**SALES_FACT:**
```sql
CREATE TABLE sales_fact AS
SELECT
    InvoiceNo,
    StockCode,
    Quantity,
    UnitPrice,
    TotalRevenue,
    CustomerID,
    TransactionDate
FROM sales_raw
```

**Result:** 395,000 clean transaction records ✅

## Appendix: SQL Validation Scripts

### Validation Script 1: Data Completeness

```sql
SELECT
    'InvoiceNo' as Column,
    COUNT(*) as Total,
    COUNT(InvoiceNo) as NonNull,
    CAST(COUNT(InvoiceNo) * 100.0 / COUNT(*) AS DECIMAL(5,2)) as Percent_Complete
FROM sales_fact
UNION ALL
SELECT 'StockCode', COUNT(*), COUNT(StockCode), 
    CAST(COUNT(StockCode) * 100.0 / COUNT(*) AS DECIMAL(5,2))
FROM sales_fact
UNION ALL
SELECT 'Quantity', COUNT(*), COUNT(Quantity), 
    CAST(COUNT(Quantity) * 100.0 / COUNT(*) AS DECIMAL(5,2))
FROM sales_fact
UNION ALL
SELECT 'CustomerID', COUNT(*), COUNT(CustomerID), 
    CAST(COUNT(CustomerID) * 100.0 / COUNT(*) AS DECIMAL(5,2))
FROM sales_fact
ORDER BY Column
```

### Validation Script 2: Referential Integrity

```sql
-- Check for orphaned StockCodes
SELECT COUNT(*) as Orphaned_StockCodes
FROM sales_fact s
WHERE NOT EXISTS (SELECT 1 FROM dim_products p WHERE p.StockCode = s.StockCode)

-- Check for orphaned CustomerIDs
SELECT COUNT(*) as Orphaned_CustomerIDs
FROM sales_fact s
WHERE NOT EXISTS (SELECT 1 FROM dim_customers c WHERE c.CustomerID = s.CustomerID)

-- Check for orphaned Dates
SELECT COUNT(*) as Orphaned_Dates
FROM sales_fact s
WHERE NOT EXISTS (SELECT 1 FROM dim_calendar c WHERE c.Date = s.TransactionDate)
```

### Validation Script 3: Revenue Calculation

```sql
-- Verify revenue calculation
SELECT
    COUNT(*) as Total_Records,
    SUM(Quantity * UnitPrice) as Calculated_Revenue,
    SUM(TotalRevenue) as Stored_Revenue,
    ABS(SUM(Quantity * UnitPrice) - SUM(TotalRevenue)) as Variance
FROM sales_fact
```
